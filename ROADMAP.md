# Roadmap

План развития по этапам. Каждому этапу соответствует epic-issue с под-задачами на GitHub.
Архитектурное обоснование — [docs/openclaw-integration.md](./docs/openclaw-integration.md).

- [x] Хронологический порядок истории в промпте

## Этап 1 — Ядро: Brain, коннекторы, единая память

Инверсия ролей: OpenClaw — мозг и память, secretary — тонкий коннектор поверхностей.

- Интерфейс `Brain` + драйвер `stateless-llm` (текущее поведение как fallback)
- Драйвер `openclaw`: сессии агента per-contact, память в workspace
- Платформо-нейтральный конверт сообщения + `capabilities`
- Персона в конфиг: матрица «платформа × поверхность», раскрытие ИИ — опция
- Identity-слой: память по персонам, склейка платформенных ID только по подтверждению владельца
- Реестр инстансов + маршрутизация (`instances.yaml`)

## Этап 2 — Control plane: управление из Telegram

- Inline-кнопки на pending-уведомлении: «Отправить сейчас / Свой ответ / Отменить»
- Draft-режим: черновик владельцу, отправка по подтверждению
- Глобальный выключатель и режимы («я свободен», «отпуск», пауза по контакту)
- Политики контактов: `auto / escalate / ignore` (VIP-список, семья — без автоответа)
- Debounce уведомлений при серии сообщений

## Этап 3 — Канал, комментарии, чат

- Ответы на комментарии канала (бот-админ в discussion group)
- Q&A в групповом чате (триггеры: упоминание/reply) + rate-limit
- Автопостинг по контент-плану, draft-режим по умолчанию
- Воронка: пост → кнопка «Спросить ассистента» → бот → продажа
- Правила памяти для публичных поверхностей (не засорять память шумом)

## Этап 4 — Мультиплатформенность

Порядок — по реалистичности API:

1. **ВКонтакте** — Callback API сообщества: ЛС, стена, комментарии (официально и бесплатно)
2. **WhatsApp** — Business Cloud API (платные диалоги, окно 24 ч)
3. **Instagram** — Meta Graph API (app review, только Business/Creator-аккаунты)

## Этап 5 — Надёжность и качество

- Тесты (vitest + supertest на DRY_RUN-режиме) — расширение текущего smoke-CI
- Ротация/TTL логов (`log-*.jsonl` содержат переписки)
- SQLite (`better-sqlite3`) вместо JSON-файлов
- Не-текстовые сообщения: голосовые через Whisper, фото — эскалация владельцу
- Реалистичность: `sendChatAction(typing)`, отметка прочитанности
- Валидация env при старте, Dockerfile для самого прокси

## Постоянно

- Проактивный дайджест владельцу: кто писал, что спрашивал, горячие лиды
- Аналитика: конверсия, среднее время ответа, топ вопросов
