Архитектурное обоснование — [docs/openclaw-integration.md](./docs/openclaw-integration.md).

- [x] Платформо-нейтральный конверт сообщения + `capabilities` — #8

- [x] Identity-слой: персоны, политики, слияние только явное (API) — #10
      (кнопка подтверждения склейки в Telegram — этап 2)

- [x] Бонус из этапа 5: Dockerfile + docker-compose, валидация env, тесты (29, node:test),
      эскалация не-текстовых сообщений

## Этап 2 — Control plane: управление из Telegram ✅ (PR #1)

- [x] Inline-кнопки на pending-уведомлении: «⚡ Сейчас / ✍️ Свой ответ / 🚫 Отменить» — #13
- [x] Draft-режим: черновик владельцу, отправка по подтверждению, «Переписать» с комментарием — #14
- [x] Глобальные режимы: `/on` `/off` («я свободен») `/vacation` + `/status` — #15
      (пауза по конкретному контакту = политика `ignore`/`escalate` кнопкой)
- [x] Политики контактов `auto / escalate / ignore` кнопками из Telegram — #16
- [x] Debounce уведомлений при серии сообщений (редактируемое уведомление) — #13

## Этап 3 — Канал, комментарии, чат

- Ответы на комментарии канала (бот-админ в discussion group)
- Q&A в групповом чате (триггеры: упоминание/reply) + rate-limit
- Автопостинг по контент-плану, draft-режим по умолчанию
- Воронка: пост → кнопка «Спросить ассистента» → бот → продажа
- Правила памяти для публичных поверхностей (не засорять память шумом)

## Этап 4 — Мультиплатформенность

Порядок — по реалистичности API:

1. **ВКонтакте** — Callback API сообщества: ЛС, стена, комментарии (официально и бесплатно)
2. **WhatsApp** — Business Cloud API (платные диалоги, окно 24 ч)
3. **Instagram** — Meta Graph API (app review, только Business/Creator-аккаунты)
