# Instagram: исследование Meta Graph API (issue #23, шаг 1)

> Статус: исследование. Решение о коннекторе — после WhatsApp (#22).
> ⚠️ Данные актуальны на начало 2026 года — перед стартом работ перепроверить
> в [официальной документации Meta](https://developers.facebook.com/docs/instagram-platform).

## Что реально доступно через API

| Возможность | Доступно? | Условия |
|---|---|---|
| DM бизнес/креатор-аккаунта | ✅ | Instagram Messaging API (через Messenger Platform) |
| DM обычного личного аккаунта | ❌ | Недоступно в принципе — API нет |
| Ответы на комментарии | ✅ | Graph API: чтение, ответ, скрытие, private reply |
| Упоминания (@mentions) | ✅ | Webhook + ответ |
| Публикация постов | ✅ | Content Publishing API (бизнес-аккаунты) |
| Stories | частично | Публикация — да; реакции на чужие — нет |

## Обязательные требования

1. **Instagram Business или Creator аккаунт**, привязанный к **Facebook-странице**
2. **Meta-приложение** с прохождением **App Review**:
   - разрешения `instagram_manage_messages`, `instagram_basic`,
     `pages_manage_metadata` (+ `instagram_manage_comments` для комментариев)
   - на ревью нужна запись экрана работающего флоу и работающий тестовый аккаунт
   - сроки ревью — от нескольких дней до недель, возможны отказы с доработкой
3. **Верификация бизнеса** (Business Verification) для продвинутого доступа
4. Webhooks: подписка на `messages`, `comments`, `mentions` — HTTPS endpoint
   с verify-token (механика как у нашего `/vk/callback`)

## Ограничения политик Meta (важно для персоны)

- **Обязательное раскрытие автоматизации**: бот должен представляться ботом —
  режим «не раскрывать ИИ» здесь запрещён политиками платформы.
  Наш persona-конфиг это уже поддерживает (`disclosure` per-surface) — для
  `instagram:*` disclosure должен быть принудительно `true` на уровне кода
- **Окно 24 часа** для ответа в DM (после — только human agent tag, 7 дней,
  по узким сценариям); проактивные сообщения первым — нельзя
- Rate limits per-app и per-user; на старте Standard Access с низкими лимитами

## Как ляжет на нашу архитектуру

Благодаря этапу 1 — обычный коннектор, ядро не трогаем:

- `src/connectors/instagram/`: webhook (verify-token GET + события POST) → конверт
  (`platform: 'instagram'`, surface `dm` | `comments`) → Brain
- Identity: `instagram:<igsid>` → персона; склейка с TG/VK — по подтверждению (есть)
- Комментарии — через публичные черновики владельцу (механика этапа 3 готова)
- `capabilities`: без typing/buttons — ядро уже деградирует корректно

## Оценка и рекомендация

| Критерий | Оценка |
|---|---|
| Техническая сложность коннектора | Низкая (≈ ВК-коннектор) |
| Организационный порог | **Высокий**: бизнес-верификация + app review |
| Ценность для типового пользователя шаблона | Средняя (нужен IG-бизнес со связанной FB-страницей) |

**Рекомендация:** делать после WhatsApp (#22 — тот же Meta-аккаунт и схожий
онбординг, но проще и полезнее для лички). Если у владельца уже есть
верифицированный Meta Business — порог резко падает, можно поднимать приоритет.
