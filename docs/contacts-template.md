# CONTACTS.md — шаблон

> Реальный файл с контактами лежит в `STATE_DIR` и в `.gitignore` — НЕ коммитится.
> Здесь только структура и пример.

## Назначение

Файл-шпаргалка по контактам. Используется в режиме «секретарь-прокси»:
- Смотрю кто пишет → подгружаю контекст → отвечаю в стиле наших отношений
- Помогает отличить клиента от незнакомца
- Хранит ID ботов, инстансов, известных проблем

## Структура

```markdown
# CONTACTS.md — Шпаргалка по контактам

## 🟢 Владелец
- **TG:** @username, id `XXX`
- **Имя, город, тон общения**
- **Контекст:** проекты под его управлением

## 💼 Клиенты <product>

### <Имя контакта>
- **TG:** @username, id `XXX`
- **Продукт/услуга:** что у них есть
- **Инстанс/инфра:** ID, сервер, порт
- **Workspace:** путь
- **Известные проблемы:** ⚠️ что висит
- **Стиль общения:** как с ним разговаривать

## 🤝 Партнёры
### <Имя>
- **Партнёрство:** что общего
- **Договорённости:** конкретно
- **События:** даты, обещания

## 👨‍👩‍👧 Семья владельца
### <Имя>, <возраст>, отношение
- **Стиль:** как обращаться

## 🐾 Питомцы (если есть)
```

## Правила обновления

**Когда обновляем:**
- Появился новый контакт → добавить в раздел
- Узнала новое о существующем (id, бот, проект, договорённость) → дописать в карточку
- Контакт переехал между категориями → переместить
- Решилась известная проблема → удалить «⚠️», обновить статус
- Что-то обсудили → записать договорённость

**Как обновляем:**
- Не спрашивать разрешения у владельца на правки в CONTACTS.md — это рабочий файл
- Маленькие правки (дата, статус) — молча
- Крупные правки (новая категория, удаление контакта) — упомянуть владельцу

**Когда проверяем:**
- Перед ответом на любое входящее сообщение от знакомого контакта
- Через RAG: семантический поиск если контакта нет в файле

## Связанные файлы

- `memory/YYYY-MM-DD.md` — daily-логи
